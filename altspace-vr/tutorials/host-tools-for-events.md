# <a name="host-tools-for-frontrow-events"></a>用于 FrontRow 事件的主机工具

本文基于你为事件启用了 FrontRow 的情况。 若要为你的事件启用 FrontRow，请参阅此文章： [使用 FrontRow 功能缩放你的受众](../faqs/scaling-audiences.md)。

如果你想要了解有关非 FrontRow 事件的主机工具，请在此处查找： [主机工具概述](host-tools-overview.md)

现在，你已将事件设置为 FrontRow，接下来我们将介绍如何使用通过 FrontRow 启用的所有出色功能！

最初创建事件时，会为该事件设置主机 "Role"。 你还可以向其他人提供主机角色，以帮助运行你的事件，他们将拥有所有相同的托管功能，然后 (在此处查看有关角色的详细信息： [事件、世界、组和 AltspaceVR](../getting-started/roles.md)) 中的角色。 主机角色授予你对专门创建的特殊特性和功能的访问权限，以使实时虚拟事件更简单、更简单。

下面是这些功能和功能的详细概述，以及在 AltspaceVR 中承载 FrontRow 事件的其他有用提示。

## <a name="host-tools-overview"></a>主机工具概述

宿主工具显示在视图字段的右下方中。 在创建事件后，可以立即使用它，因此，你可以随时输入事件 (事件开始时间不适用于事件) 的主机或仲裁者，并在 showtime 之前获得工具。 利用主机工具，你可以快速单击访问 (运行 FrontRow 事件所需的内容，) 、向受众发送静音和消息传递、自行切换你的阶段阻止程序、megaphoning，允许你的来宾 "提出问题" 或 "提供反馈"，以及从 FrontRow 房间移动到房间等。

!["主机工具" 菜单打开，其中突出显示了可用操作](images/host-front-row-img-01.png)

*如果看到 "中转到无线" 按钮，则可将事件设置为 "FrontRow"。*

### <a name="toggle-mute-all"></a>切换 Mute-All

通过对您的与会者进行静音，您可以限制混乱的声音，并使 FrontRow 事件中的东西保持良好和安静。 如果宿主允许，或当主机离开空间时，静音函数将持续的时间。 当与会者进入事件或尝试取消静音时，与会者将收到一条屏幕消息，让他们知道主机已将其静音。 当主机 unmutes 时，与会者将收到一条消息，告知他们已 unmuted。

### <a name="message-all"></a>全部消息

主机可以将消息发送给在其 FrontRow 事件中出现的所有与会者。 使用我们的任何快速笔记或编写原始文档。 发送后，每个人都将收到一条弹出消息。

### <a name="amplify-my-voice"></a>放大我的语音

主机可以放大其麦克风，以确保前行事件中的每个人都能听到您的意思，而不考虑距离。

### <a name="go-on-air"></a>上网

具有宿主或表示器角色的任何人都可以查看此按钮。 单击 "无线" 会将头像 "镜像" 到所有 FrontRow 房间。 一旦你进行了 "镜像"，你所做的一切就会出现在 FrontRow 事件的所有实例中。

作为 FrontRow 事件的整体，这一点非常重要。 你必须是无线的，主机工具才能在所有 FrontRow 房间上正常工作，否则，它们只能在你所在的房间内工作。

> [!NOTE]
> 如果需要重新加入事件，请转到 Campfire 并返回，失去 Internet 连接，从房间移到房间，当您回来时，您将不会在无线上。 请在返回后再次单击 "继续"。

### <a name="toggle-stage-blocker"></a>切换阶段阻止

确定与会者可以参加的时间和位置。 选择占用，在 dance 参与方的舞台上走走并邀请每个人。

> [!IMPORTANT]
> 并非所有世界都包含阶段阻止程序，因此请明智地选择事件模板。 创建你的官方活动之前，先熟悉我们的测试事件的事件模板。

## <a name="guest-panel"></a>来宾面板

这使你可以跟踪事件 "房间" 中的所有人。 在 FrontRow 事件中有多个房间，来宾面板仅向你显示你当前所在房间中的来宾。 可以通过单击 "所有/打开 Mic/主机/好友/无线" 来筛选显示的内容。 它是一个交互式列表，可提供房间内用户的多页细目细目。

尝试查找具有不良音频的用户？ 单击 "打开 Mic" 查看 unmuted 的用户。 想要确保 FrontRow 事件中所需的所有人都已镜像？ 单击 "无线" 查看当前在无线和已镜像的所有人。 

### <a name="moderation-tools"></a>裁决工具

作为事件主机，你负责管理其他人的体验。 若要高效地执行此操作，你和你的审查人员需要稳健的审核工具。 作为 FrontRow 事件的宿主，你可以为中断性人员静音，开始规则 (暂时和/或事件的持续时间) ，甚至在事件期间为其他人分配仲裁者/主机角色，以帮助管理受众。

您可以通过单击其名称旁边的扬声器图标或其他选项，单击来宾面板右侧的汉堡菜单，立即从 "来宾" 面板中将来宾静音：

![在来宾面板中选择了单个来宾项目](images/host-front-row-img-03.png)

*单击汉堡菜单以打开所有 "裁决工具" 选项。*

![突出显示了裁决工具选项的单个来宾项目](images/host-front-row-img-04.png)

现在，你可以使用 FrontRow 托管幻，并能够在你的与会者上使用这些功能：
    * **消息** -通常，你需要成为好友才能向其他用户发送消息，但作为主机，你可以发送/接收你事件中任何人的消息。
    * 为事件中的所有 **人静音。**
    * **扩音器** -让他们能够放大我的语音。
    * **在无线上** -你可以 "镜像" 此来宾，这将使 FrontRow 事件的所有聊天室中的所有人都可以看到并通知其虚拟机。
    * **阶段访问** -让他们能够完成任何阶段阻止程序并进入阶段区域。
    * **使主机** -在 FrontRow 事件中为其提供主机角色。 动态添加 Co-Host，确保它们在无线上运行，并可帮助你运行某些功能，或者将其传送到其他房间来检查东西。 每个人是否都在辅助房间内显示？ 你的演示文稿是否跨所有活动的会议室同步？ 当他们在其他房间内时，让他们进入飞机，你将能够将头像与房间间的头像交谈！
    * **做出审查** -为他们提供前行事件中的 "仲裁者" 角色，以帮助你进行大规模控制。 版主将有权访问有限的主机工具，因此他们可以从房间移动到房间来检查东西，但无需无线访问。 
    * **启动** 规则-在事件发生期间，暂时或发生事件。

## <a name="participation-panel"></a>参与面板
 
您可以使用此信息帮助您与事件与会者交互。 你可以设置一个 Q&，形成一个用于打开的 Mic 的队列，或确保你正在交谈的来宾也是镜像的，以便每个人都可以听到你在所有 FrontRow 房间之间相互交谈：

![选择了 "参与" 选项卡时打开的主机面板](images/host-front-row-img-05.png)

加入面板旨在帮助您管理用户的交互和参与。 在这里，你可以在 "允许举手" 选项上切换，以便解锁观众参与情况。 **在 FrontRow 事件中，需要先将其激活，否则此按钮将显示为灰色。**

激活 "允许举手" 功能后，FrontRow 事件中的每个人都将出现 "举手" 按钮，位于其字段视图的右下方：

![参与面板中突出显示的举手功能](images/host-front-row-img-06.png)

*单击 "举手" 会将来宾置于您控制的队列中。*

当来宾单击此按钮时，它们将显示在你的参与面板中：

![已突出显示队列中的 "手动抛出" 的参与面板](images/host-front-row-img-07.png)

为帮助您跟踪，如果有客人等待呼叫，加入面板甚至会亮起红色通知，并且它将告诉您每个来宾的等待时间。

现在，你可以通过单击 "无线 + 扩音器" 按钮对其进行调用，这会使按钮在激活后变为蓝色。 此操作会向手中发送两条消息，以确认是否要将其麦克风取消静音，并且 "主机已呼叫你，并使你变得更大"。 接受后，该按钮将变为蓝色，这会将其放在无线 "镜像" 上，并放大我的语音 "Megaphoned"，以查看和听到！

完成与它们的交互后，首先再次单击蓝色按钮以使其进入 "无线" 状态，按钮将不再为蓝色，然后单击 "X" 以从列表中将其关闭。 现在，你已准备好获得幸运的 contestant 号，等等： ) 

你还可以向其授予临时访问权限，或单击右侧的 "全部取消" 按钮，以完全清除右手 Raisers 列表并从头开始。

## <a name="room-panel"></a>房间面板

快速了解在 FrontRow 事件中管理会议室所需的信息/按钮：

![从主机面板打开的会议室面板选项卡](images/host-front-row-img-08.png)

*请注意，这表示你 **)  (**，这是你当前所在的空间。*

房间包含用于重置空间、删除空间、重新分布用户和传送的按钮：

![房间面板中突出显示的 "重置空间" 按钮](images/host-front-row-img-09.png)

将在你当前所在的房间中执行 "重置空间"。 你不能同时为所有房间重置空间，你需要移动到每个房间来进行重置。

![房间面板中突出显示的 "删除空间" 按钮](images/host-front-row-img-10.png)

永久删除当前所在的聊天室。 已删除房间中的每个人都将自动重新加入事件并加入不同的房间。

![房间面板中突出显示的 "重新分发用户" 按钮](images/host-front-row-img-11.png)

有时，在 FrontRow 事件已开启后，前几个房间几乎都处于容量，但最后一个房间或两个房间只能有少量的来宾。 单击此按钮后，就可以选择在其当前房间内或转到更多的空间。

> [!NOTE]
> 你不必旅行到想要重新分发的房间，只需单击房间面板，就可以在任何房间上单击 "重新发布用户" 按钮。 不要在同一房间内尝试执行此操作的次数过多，可能只需要几个人，也可能不想让收到的消息进入请求移动。

![在房间面板中显示突出显示的按钮](images/host-front-row-img-12.png)

这就是您从一个前置行房间到达另一个房间的方式。 版主需要使用这种方法从房间转到房间来查看 crowds。 在 rehearsals 期间，你可以在主房间内有一台主机，并让你的其他房间进入其他房间，以查看所有内容的外观。 其他房间中的所有内容是否同步，并看起来就像是在主房间中一样？ 如果帮助测试的每个人都具有主机角色，则他们也可以传播到其他房间，外出，你将能够观看和倾听每个人，使你能够通过这种方式进行通信。

> [!IMPORTANT]
> 尝试转到空房间时，会收到一条警告： "打开其他房间？ Teleporting 到空房间还会将其打开。 " 在 **实际的 FrontRow 事件期间，你和你的活动人员必须在事件开始时间之前的五分钟内全部处于主房间中** (门在事件开始时间之前打开3分钟，因此，为了安全起见，请将其设置为5分钟) 否则，将不会像预期那样向事件的主要房间填写记录，这将为其余事件引发排序。
