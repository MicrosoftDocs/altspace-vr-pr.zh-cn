# <a name="host-tools-for-frontrow-events"></a>FrontRow 事件的主机工具

本文基于为事件启用了 FrontRow。 若要为活动启用 FrontRow，请参阅以下文章： [使用 FrontRow 功能缩放受众](../faqs/scaling-audiences.md)。

若要了解非 FrontRow 事件的主机工具，请在此处查看： [主机工具概述](host-tools-overview.md)

将事件设置为 FrontRow 后，让我们了解如何使用通过 FrontRow 启用的所有出色功能！

最初创建事件时，为该事件设置了主机"角色"。 还可以将"主机"角色授予其他想要帮助运行事件的人，他们拥有与托管功能相同的托管功能 (在此处查看有关角色（事件、世界、组和 [AltspaceVR](../getting-started/roles.md)) 中的角色） 。 "主机"角色授予你访问专为简化实时虚拟事件托管而创建的特殊特性和功能。

下面详细介绍了这些功能、功能，以及用于托管 AltspaceVR 中的 FrontRow 事件的其他有用提示。

## <a name="host-tools-overview"></a>主持人工具概述

"主机工具"显示在视场右下侧。 创建事件后立即可用，因此，每当 (事件开始时间不适用于事件) 的主机或审查器时，都可以输入事件，并熟悉显示时间之前的工具。 使用主机工具，可以快速单击访问运行 FrontRow 事件) 所需的 On-Air (、静音和向受众传送消息、打开和关闭阶段阻止程序、自己大步操作、允许来宾"举手"提问或提供反馈、从 FrontRow 房间移动到房间等。

!["主机工具"菜单打开，突出显示了可用操作](images/host-front-row-img-01.png)

*如果看到"打开 AIR"按钮，可以告知事件已设置为 FrontRow。*

### <a name="toggle-mute-all"></a>切换Mute-All

通过使参与者静音，可以限制分散注意力的声音，让 FrontRow 活动中的一切保持友好和静默。 只要主机允许，或者当主机离开空间时，互斥函数将持续一段时间。 当参与者进入活动或尝试取消静音时，参与者将收到一条屏幕消息，让他们知道他们已被主机静音。 当"主机"取消静音时，参与者将收到一条消息，让他们知道他们也是未静音的。

### <a name="message-all"></a>全部消息

主机可以跨 FrontRow 事件向所有参与者发送消息。 使用我们的任何快速说明或编写原始说明。 发送后，每个人都会收到弹出消息。

### <a name="amplify-my-voice"></a>增强我的语音

主机可以放大麦克风，以确保前行活动中的每个人都会听到你说话，而不管距离如何。

### <a name="go-on-air"></a>上车

具有"主机"角色的任何人都可以看到此按钮。 单击"上线"将头像"镜像"到所有 FrontRow 房间。 "镜像"后，所有 FrontRow 事件实例中都会看到你所说的和所做的一切。

在整个 FrontRow 事件上是 On-Air 非常重要。 你必须是 On-Air，主机工具才能在所有 FrontRow 房间中正常运行，否则它们只能在你位于的聊天室中工作。

> [!NOTE]
> 如果需要重新加入活动，请转到"Campfire"并返回，失去 Internet 连接，从"房间"移动到"房间"，当你回来时，你将不是"空"。 请务必在返回后再次单击"上一步"。

### <a name="toggle-stage-blocker"></a>切换阶段阻止程序

确定参与者可以站在哪里。 选择让聚焦、轮次呈现，或邀请每个人上场参加组会。

> [!IMPORTANT]
> 并非所有世界都包含阶段阻止程序，因此请明智选取事件模板。 在创建官方活动之前，请熟悉测试事件的事件模板。

## <a name="guest-panel"></a>来宾面板

允许你跟踪事件"房间"中的每个人。 在 FrontRow 活动中，有多个房间，来宾面板只显示你当前位于房间中的来宾。 可以通过单击"全部"/"打开麦克风"/"主机"/"好友"/"空"来筛选显示内容。 它是一个交互式列表，提供房间中谁在多页细分。

尝试查找音频错误的人？ 打开麦克风以查看谁未静音。 想要确保所需的每个人都在 FrontRow 事件中镜像？ 选择"On-Air"以查看当前为"On-Air"和"Mirrored"的所有人。 

### <a name="moderation-tools"></a>审核工具

作为事件主机，你负责管理其他人的体验。 为了高效审查，你和审查器需要可靠的审查工具。 作为 FrontRow 事件的主机，你可以将干扰性个人静音，暂时或在你的事件) 期间从事件 (中启动规则破坏者，甚至可以在活动期间将审查器/主机角色分配给其他人，以帮助管理受众。

通过单击来宾姓名旁边的说话人图标，或者选择来宾面板右侧汉堡菜单，即可将其从来宾面板中静音：

![在来宾面板中选择了单个来宾项](images/host-front-row-img-03.png)

*单击汉堡菜单打开所有"审核工具"选项。*

![选中的单个来宾项，并突出显示了审核工具选项](images/host-front-row-img-04.png)

现在，你能够使用 FrontRow 托管 Magic，并能够在与会者上使用以下功能：
    * **消息** - 通常，你需要是好友才能向其他用户发送消息，但作为主机，你可以与活动中的任何人发送/接收消息。
    * **静音** - 对活动中的每个人将其静音。
    * **Megaphone** - 让他们能够放大"我的语音"。
    * **On-Air** - 可以"镜像"此来宾，这样，FrontRow 活动中所有房间的每个人都可以看到和听到其头像。
    * **阶段访问** - 让他们能够通过任何阶段阻止程序并进入阶段区域。
    * **使主机** - 在 FrontRow 事件中为它们提供主机角色。 添加Co-Host，确保它们位于"On-Air"中，它们可以帮助你运行内容或让他们前往其他房间检查内容。 是否每个人都在上空并显示在辅助房间中？ 你的演示文稿是否在所有活动房间之间同步？ 当他们位于另一个房间时，请让他们上空，你将能够从房间到房间与头像对话！
    * **使审查** 器 - 在 Front Row 活动中为它们提供审查器角色，以帮助你控制群。 审查器将有权访问有限的主机工具，因此他们可以从房间移动到房间来检查内容，但无需进行"打开"操作。 
    * **Kick** - 暂时或在事件期间将规则断路器从事件中启动。

## <a name="participation-panel"></a>参与面板
 
可以使用此面板来帮助与活动参与者进行交互。 可以&设置问答，形成一个队列，用于打开麦克风或确保正在与交谈的来宾也进行镜像，以便每个人都可以听到你在所有 FrontRow 房间之间相互交谈：

![主机面板打开，已选择"参与"选项卡](images/host-front-row-img-05.png)

参与面板旨在帮助你管理受众的交互与参与。 可在此处切换"允许手部引发"选项，以解锁受众参与。 **在 FrontRow 事件中，需要先使用 On-Air 才能激活此功能，否则此按钮将显示为灰色。**

激活"允许引发手部"函数后，FrontRow 事件右下角部分会显示一个"举手"按钮：

![在参与面板中突出显示的"引发手部"函数](images/host-front-row-img-06.png)

*单击"举手"会将来宾置于你控制的队列中。*

当来宾单击此按钮时，他们会显示在参与面板中：

![参与面板打开，来宾在队列中突出显示了手部](images/host-front-row-img-07.png)

为了帮助你进行跟踪，当你有来宾等待被调用时，参与面板甚至会发出一条红色通知，并告知每个来宾已等待了多久。

现在，可以通过单击"On-Air + Megaphone"按钮来调用它们，激活后，此按钮将变成蓝色。 此操作将向手部引发程序发送两条消息，以确认他们希望取消麦克风静音，并且"主机已调用你并提高了音量。" 一旦他们接受，按钮将变蓝色，这会将它们置于无线"镜像"和"放大我的语音""Megaphoned"中，供所有用户观看和听到！

完成与它们交互后，首先再次单击蓝色按钮以将其关闭，该按钮将不再为蓝色，然后单击"X"以从列表中消除它们。 现在，你已准备好参加比赛，第二号获胜者，等等，等等：) 

还可以为它们提供临时阶段访问，或单击右侧"全部消除"按钮，以完全清除手部引发程序列表并开始更新。

## <a name="room-panel"></a>房间面板

快速查看在 FrontRow 事件期间管理房间所需的信息/按钮：

![从主机面板打开"房间面板"选项卡](images/host-front-row-img-08.png)

*请注意 **， (你** 位于) ，即当前位于房间。*

房间具有"重置空间"、"删除空间"、"重新分发用户"和"Teleport There"按钮：

![房间面板中突出显示的"重置空间"按钮](images/host-front-row-img-09.png)

将执行当前房间中的"重置空间"。 不能一次重置所有房间的空间，需要前往每个房间进行重置。

![房间面板中突出显示的"删除空间"按钮](images/host-front-row-img-10.png)

永久删除当前使用的空间。 已删除房间中的每个人都将自动重新加入活动并加入其他房间。

![房间面板中突出显示的"重新分发用户"按钮](images/host-front-row-img-11.png)

有时，在 FrontRow 活动打开一段时间后，前几个房间几乎可以容纳，但最后一两个房间每个房间可能只有少量来宾。 单击此按钮，该房间中的来宾可以选择留在当前房间或进入更填充的房间。

> [!NOTE]
> 不需要前往要重新分发的会议室，可以从"房间"面板右键单击任何房间上的"重新分发用户"按钮。 尝试不要多次向同一房间进行此操作，该房间中的来宾可能只想与一些人员在一起，并且可能不希望不断收到要求移动房间的消息。

![房间面板中突出显示了 Teleport there 按钮](images/host-front-row-img-12.png)

这就是从一个 Front Row Room 到另一个前行房间的出行方式。 审查器将希望使用此设备从房间到房间四处弹跳，以检查人流。 在排演期间，可以在主房间中设置一个主机"On-Air"，让其余工作人员转到其他房间，查看一切的外观。 其他房间中是否一切同步，看起来就像在主房间中一样？ 如果帮助测试的每个人都具有"主机"角色，他们还可以前往其他房间，转到"On-Air"，你将能够看到和听到每个人，以便你能够这样跨房间进行通信。
